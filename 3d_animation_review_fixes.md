# 3Dアニメーション確認・修正点

作成日: 2026-06-11

## 対象

3D実戦体験モード `/#/experience` の麻雀卓アニメーションを確認しました。

主な確認対象:

- `src/core/sim/kyokuSim.ts`
- `src/core/sim/scenarioEvents.ts`
- `src/core/sim/boardLayout.ts`
- `src/features/experience/timeline/compileTimeline.ts`
- `src/features/experience/timeline/captionScript.ts`
- `src/features/experience/timeline/cameraScript.ts`

画面確認は Vite 開発サーバー `http://127.0.0.1:5173/#/experience` で行いました。

## 結論

ユーザー指摘のうち、リーチ宣言牌の向きは実装上のバグとして優先修正が必要です。

配牌とドラ表示はロジック上は成立していますが、現状の演出では麻雀卓としての見え方が弱く、説明字幕と実際の牌の動きにズレがあります。特に配牌は「4枚ずつ取る」「親だけ14枚で始まる」という説明に対して、アニメーション状態は最終手牌へ直接移動しているため、横積み・山からの取り出し感が不足しています。

## 修正点

### 1. リーチ宣言牌が横向きではなく立って見える

優先度: 高

該当箇所:

- `src/core/sim/boardLayout.ts:115`
- `src/core/sim/boardLayout.ts:124`

現状:

```ts
return seatLocalPose(seat, localX, LYING_Y, localZ, [
  -Math.PI / 2,
  riichiTilt ? Math.PI / 2 : 0,
  0,
])
```

通常の河牌 `[-Math.PI / 2, 0, 0]` は牌面の法線が上を向きます。一方、リーチ牌 `[-Math.PI / 2, Math.PI / 2, 0]` は牌面の法線が横向きになり、画面上でも横倒しの牌ではなく縦に立った板のように見えます。

修正方針:

- リーチ牌はテーブル面内で90度回す必要があるため、Y軸ではなくZ軸回転で表現する。
- 例: `[-Math.PI / 2, 0, riichiTilt ? Math.PI / 2 : 0]`
- `seatLocalPose` が座席角度をY軸へ加算しているため、座席ごとの向きとリーチ牌の面内回転が干渉しないか確認する。

受け入れ条件:

- リーチ宣言牌の牌面が上を向いたまま、河の通常牌に対して90度横向きになる。
- 上家・下家・対面のリーチでも、牌が立ち上がらない。
- `boardLayout.test.ts` に「リーチ牌の法線が上向きのまま」という姿勢テストを追加する。

### 2. 配牌が「4枚ずつ横積みで取る」動きに見えない

優先度: 高

該当箇所:

- `src/core/sim/kyokuSim.ts:159`
- `src/core/sim/kyokuSim.ts:161`
- `src/core/sim/scenarioEvents.ts:263`
- `src/features/experience/timeline/compileTimeline.ts:122`

現状:

- シミュレーション上は `kyokuSim.ts` で4枚ずつ3回、最後に1枚ずつ配る順序を作っている。
- ただし `deal` イベントは最終的な `hands` だけを渡す。
- `applyEvent` では各牌を直接 `hand` ゾーンへ置き、すぐ `withResortedHand` でソートする。
- そのため、アニメーションでは「山から4枚単位の横積みブロックを取る」途中状態が表現されない。

修正方針:

- `deal` を1イベントで完結させず、途中状態を持てるようにする。
- 候補1: `TileZone` に `dealBlock` または `handStaging` を追加し、4枚単位で横積み・仮置きする。
- 候補2: `ScenarioEvent` を `dealBlock`、`dealSingle`、`dealSort` のように分ける。
- 最終ソートは配牌完了後に行い、配牌中は山から取った物理順を保つ。

受け入れ条件:

- 配牌中、各家が4枚ずつ3回受け取る動きが画面上で分かる。
- 4枚はバラバラに飛ばず、2列x2段または横並びのまとまりとして見える。
- 配牌完了後にだけ手牌が整列する。

### 3. 親の14枚目が配牌イベント内で表現されていない

優先度: 中

該当箇所:

- `src/core/sim/kyokuSim.ts:161`
- `src/core/sim/kyokuSim.ts:169`
- `src/features/experience/timeline/captionScript.ts:112`

現状:

字幕では「親はさらに1枚多く取るため、親は14枚、子は13枚でスタート」と説明している。一方、`deal` イベントでは全員13枚まで配り、親の14枚目は次の `draw` イベントとして扱われる。

これはゲーム進行としては「親の第一ツモ」として処理できるものの、配牌説明と3D演出の見た目が一致しません。

修正方針:

- 説明に合わせるなら、`deal` イベントに親の14枚目も含める。
- 進行モデルを維持するなら、字幕を「配牌後、親が最初の1枚を取って第一打します」のように修正する。
- 3D教材としては、親の14枚目を配牌演出内で見せる方が分かりやすい。

受け入れ条件:

- 配牌チャプター終了時点で、親だけ14枚、子は13枚に見える。
- その後、親はツモ動作なしで第一打に入る、または字幕が現在の挙動と一致する。

### 4. ドラ表示牌は状態上はめくられているが、卓上で分かりにくい

優先度: 中

該当箇所:

- `src/core/sim/scenarioEvents.ts:141`
- `src/core/sim/scenarioEvents.ts:273`
- `src/core/sim/boardLayout.ts:86`
- `src/features/experience/timeline/compileTimeline.ts:122`

現状:

- `DORA_INDICATOR_DEAD_INDEX = 4` でドラ表示牌を選んでいる。
- `deal` イベント適用時に `deadWall` の対象牌を `faceUp: true` にしている。
- HUDには「ドラ表示 八萬」が表示されている。
- ただし3D卓上では、ドラ表示牌がどこでめくられたかが目立たない。カメラが手牌や全体に寄るため、ユーザーには「ドラが捲られていない」と見えやすい。

修正方針:

- `deal` とは別に `doraReveal` イベントを追加し、ドラ表示牌だけを明示的に回転させる。
- カメラを王牌へ寄せる短いショットを入れる。
- ドラ表示牌に一瞬だけハイライト、光、または小さな注目エフェクトを付ける。
- HUDだけでなく卓上の対象牌が分かるようにする。

受け入れ条件:

- 配牌後、王牌上のドラ表示牌が明確に表向きになる。
- HUDのドラ表示と卓上の表示牌が一致して見える。
- スクリーンショットだけでも、どの牌がドラ表示牌か判別できる。

### 5. ドラ表示牌の位置が実卓の王牌表現として十分か再確認する

優先度: 低から中

該当箇所:

- `src/core/sim/scenarioEvents.ts:141`
- `src/core/sim/boardLayout.ts:160`

現状:

`deadWall` は `wallSlotSequence` の末尾14枚として配置され、`DORA_INDICATOR_DEAD_INDEX = 4` を表示牌にしている。上段3幢目相当を狙っているように見えますが、開門位置・山の進行方向・上段/下段の対応が実卓表現として正しいかは、テストでは検証されていません。

修正方針:

- 王牌14枚の見た目を「7幢2段」として明示的に扱う。
- ドラ表示牌の位置を「王牌の端から3幢目の上段」としてテスト化する。
- `wallSlotSequence` の消費順と、表示上の王牌位置の対応をコメントまたはヘルパー名で明確にする。

受け入れ条件:

- サイコロ値が変わっても、ドラ表示牌が王牌内の期待位置に出る。
- `boardLayout.test.ts` にドラ表示牌スロットの位置検証がある。

## 追加したいテスト

- `riverTilePose` のリーチ牌について、牌面法線が上向きであることを検証する。
- `deal` チャプター後の牌数が、親14枚・子13枚、または字幕と一致することを検証する。
- ドラ表示牌が `faceUp: true` になり、対応する3D姿勢が face-up であることを検証する。
- `compileTimeline` で `doraReveal` を追加する場合、配牌後に必ずドラ表示チャプターまたはクリップが生成されることを検証する。

## 確認メモ

- Vite 開発サーバーで `/#/experience` を表示し、配牌・リーチ・和了チャプターを確認しました。
- リーチチャプターでは、宣言牌が横向きではなく縦に立ったように見えることを確認しました。
- HUD上のドラ表示は出ていますが、卓上のドラ表示牌は視認性が低く、教材としては明示的なめくり演出が必要です。
