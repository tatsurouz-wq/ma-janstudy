# 麻雀ルール照合レポート

作成日: 2026-06-11

## 対象

麻雀学習アプリ「雀学」の教材本文、クイズ問題、点数計算ロジックを、README に記載された採用ルールと一般的なリーチ麻雀ルールに照らして確認しました。

確認対象の主なファイル:

- `README.md`
- `src/content/lessons/lesson01.ts` から `lesson08.ts`
- `src/content/quizzes/*.ts`
- `src/core/score/*.ts`
- `src/core/score/yaku/*.ts`
- `src/core/hand/*.ts`
- 関連テストファイル

## 採用ルール

README 上のデフォルトルールは以下です。

- 喰いタンあり
- 後付けあり
- 赤ドラ各1枚
- 一発・裏ドラあり
- 切り上げ満貫なし
- 平和ツモ20符
- 七対子25符
- 連風牌の雀頭2符
- ダブル役満なし
- 数え役満あり

この前提に対して、点数計算・符計算・役判定はおおむね整合していました。

## 発見した不整合

### 1. 配牌枚数の説明が親の扱いを省略していた

該当箇所:

- `src/content/lessons/lesson03.ts`

元の教材では「各プレイヤーは13枚の手牌からスタート」と説明していました。子は13枚で正しいですが、親は最初だけ14枚を持ち、第一打から始まります。

また、親に関するクイズの誤答選択肢に「配牌が14枚もらえる」が含まれていました。これは親については事実として正しいため、誤答として置くとルール上の混乱を招きます。

修正内容:

- 「子は13枚、親は14枚から始まる」という説明に変更
- 進行順のラベルも「配牌を受け取る（子は13枚、親は14枚）」へ変更
- 親クイズの誤答選択肢を「最初の配牌が12枚になる」に変更

### 2. 「役満（13翻〜）」の表現が数え役満と役満役を混同しやすかった

該当箇所:

- `src/content/lessons/lesson08.ts`

満貫以上の説明で「役満（13翻〜）」と書かれていました。採用ルールでは数え役満ありなので点数としては正しいですが、国士無双・四暗刻などの役満役と、通常役とドラの合計で到達する数え役満は別概念です。

修正内容:

- 「役満（13翻〜）」を「数え役満（13翻以上）」に変更

## 確認したが問題なしと判断した主な点

- 平和ツモ20符固定の扱い
- 七対子25符固定の扱い
- 連風牌雀頭2符のデフォルト設定
- 切り上げ満貫なしの点数処理
- ドラ表示牌の循環処理
- リーチ時のみ裏ドラを数える処理
- ドラのみでは和了不可とする処理
- 喰いタンなし設定時に鳴きタンヤオを無効化する処理
- 四暗刻のロン双碰和了を四暗刻不成立とする処理
- 二盃口と一盃口の重複採用を避ける処理
- 役満複合を `stackedYakuman` 設定で制御する処理

- 5 test files passed
- 165 tests passed

型チェックも成功しました。

```bash

```

注意点:

全体テスト `npm run test` は一度成功しましたが、修正後の再実行では Vitest worker の起動タイムアウトが発生しました。失敗内容は複数 worker の応答待ちタイムアウトで、今回の教材修正や麻雀ロジックの失敗ではありません。関連範囲のテストは個別実行で通過しています。

## 結論

教材・クイズ・点数計算ロジックを確認した結果、重大な採点ロジックの誤りは見つかりませんでした。

ルール説明として修正すべき点は、親の配牌枚数の扱いと、13翻以上を数え役満として明記する表現の2点でした。どちらも修正済みです。
